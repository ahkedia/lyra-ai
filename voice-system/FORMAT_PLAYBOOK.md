# Format Playbook — X Content Templates

Templates for each content format. The packaging rule (from plan section 6):
- **Timely and narrow** → single post
- **Mechanism + example + implication** → thread
- **Enduring explanatory value** → long-form essay thread
- **Strong metaphor/framework** → doodle cover or explainer

---

## Format 1: X Micro-Post (single tweet)

**Use when:** One sharp observation, reaction, or take. No arc needed.

**Structure:**
```
[claim or observation — max 240 chars]
[optional: 1 supporting line or contrast]
```

**Good examples of this shape:**
- Lead with the claim. Don't wind up.
- End without a question or CTA. The thought is enough.
- 120-200 chars is ideal. Under 100 can work if the thought is sharp.

**What to avoid:**
- Don't pad to fill the character limit
- Don't end with "thoughts?" or a hashtag

---

## Format 2: X Thread (5-12 posts)

**Use when:** Idea has a real arc: mechanism + example + implication. Needs room.

**Structure:**
```
Post 1 (hook): [sharp claim or counterintuitive observation]
Post 2: [what most people think / the conventional view]
Post 3: [why that's wrong or incomplete — the turn]
Post 4-N: [evidence, examples, mechanism, specifics]
Post N-1: [so what — implication]
Post N: [one thing to carry away — not a CTA]
```

**Hook rules:**
- First post must work standalone (it's what people share)
- No "a thread 🧵" as the only first post — the hook IS the first post
- The turn (post 3) is where you earn the reader

**Thread mechanics:**
- Posts 2-N posted as replies to post 1
- 500ms delay between each post (enforced by publisher)
- Optimal: 6-9 posts. 12 is the hard ceiling.

---

## Format 3: X Long-Form Essay Thread

**Use when:** Idea has enduring explanatory value. This is the "essay" format that replaces long-form blog posts. Published as a structured multi-part thread on X.

**Structure:**
```
Post 1 (title): [essay title + one-line premise]
Post 2 (problem): [what's broken or underappreciated]
Post 3-4 (mechanism): [why it works the way it does]
Post 5-7 (evidence): [career examples, data, specifics — this is the Akash POV section]
Post 8-9 (implications): [what this means for people in the space]
Post 10 (close): [one thing to carry away — concrete, not inspirational]
```

**Essay rules:**
- Must have a clear thesis. Not a "collection of thoughts."
- Evidence section should use first-hand examples (career, projects, observations)
- Wiki retrieval is mandatory for this format — no essay without grounded evidence
- canonical_url = URL of post 1 after publishing

**Pilot:** "Growth in the AI Era" (Phase 3)

---

## Format 4: Doodle Cover Post

**Use when:** The idea has a strong visual metaphor, diagram, or framework that makes it more legible as an image than as text.

**Structure:**
```
Post 1: [image] + [one-line caption that adds to the image, doesn't just describe it]
[optional thread follow-up]
```

**Doodle brief triggers:**
- Framework with 2-4 components → diagram
- Before/after contrast → side-by-side
- Process or pipeline → flow diagram
- Tension between two things → Venn or axis

**Visual style reference:** lyra-ai/voice-system/DOODLE_STYLE_GUIDE.md (Phase 4)

## The Packaging Decision

Before writing any draft, classify the idea:
1. Does it have a mechanism to explain? (thread or essay)
2. Does it have first-hand evidence worth sharing? (essay)
3. Does it have a visual metaphor? (doodle)
4. Is it a one-liner worth sharing now? (micro-post)

If none of the above: hold it. Not every idea is a post.
